# Systematic Capability Benchmarking of Frontier Large Language Models for Offensive Cyber Tasks

## Abstract (426 words, expected was 100-500 words):

Frontier Large Language Models (LLMs) increasingly demonstrate advanced reasoning, planning, and tool-use capabilities that extend beyond natural language tasks into complex technical domains, including cybersecurity. Executive Order 14110 on the Safe, Secure, and Trustworthy Development and Use of Artificial Intelligence explicitly highlights the need to assess whether frontier AI systems could meaningfully enhance the capability of malicious cyber actors. However, cybersecurity is inherently dual-use: the same capabilities that enable defensive automation may also increase offensive risk. Systematic, controlled evaluation methodologies are therefore required to quantify cyber task performance without facilitating real-world harm.
This work presents a comprehensive engineering study of offensive cyber task performance across frontier LLMs using a structured multi-agent evaluation architecture deployed within controlled containerized environments. We operationalize offensive capability measurement through the NYU CTF Bench, a standardized dataset of 200 Capture The Flag (CTF) challenges spanning cryptography, forensics, reverse engineering, binary exploitation, web security, and miscellaneous domains. Each challenge emulates real-world attack reasoning tasks while remaining confined to a sandboxed environment.

Our evaluation framework incorporates hierarchical task decomposition, planner–executor role separation, dynamic prompt generation, and tool-integrated execution within Docker-based Linux environments. We extend prior approaches by systematically investigating five engineering dimensions that influence cyber task performance: (1) infrastructure configuration (baseline Ubuntu versus penetration-testing-enhanced Kali Linux environments); (2) category-specific versus generic prompting strategies; (3) cross-model benchmarking of frontier systems including GPT-5 variants, Claude 4.6 variants, Gemini 3 Pro, LLaMA, Mistral, DeepSeek, and other contemporary models; (4) heterogeneous model pairing strategies that balance reasoning capacity and execution cost; and (5) statistical reproducibility through multi-trial variance analysis.

Each configuration is executed across multiple independent trials to quantify run-to-run variability. Independent variables include model selection, environment configuration, architecture assignment, and prompt strategy. Dependent metrics encompass solve rate, token consumption, API cost, reasoning rounds, error frequency, and cost-per-solved challenge. This enables multidimensional cost–performance frontier analysis and infrastructure–architecture interaction modeling.

Initial exploratory trials, informed by prior multi-agent cybersecurity benchmarks, suggest that environment configuration and model pairing strategies may influence performance, particularly in tooling-dependent challenge categories such as web and forensics. Early indications point toward potential cost–performance advantages from heterogeneous model assignments. Comprehensive multi-trial analysis is currently underway to quantify variability across runs and to establish reproducibility benchmarks for frontier AI cyber task evaluation.

This study contributes one of the most comprehensive cross-model engineering evaluations of LLM cyber task performance to date. By framing offensive capability measurement as a controlled systems-testing problem, we provide empirical insights relevant to AI risk assessment, secure model deployment, and policy-driven evaluation mandates while maintaining strict containment within benchmarked environments.
